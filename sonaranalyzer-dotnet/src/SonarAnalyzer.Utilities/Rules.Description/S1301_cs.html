switch statements and expressions are useful when there are many different cases depending on the value of the same expression.

For just one or two cases however, the code will be more readable using if statements or the ternary operator instead.

Noncompliant Code Example

switch (variable)
{
  case 0:
    doSomething();
    break;
  default:
    doSomethingElse();
    break;
}

var foo = variable switch
{
  0 => doSomething(),
  _ => doSomethingElse(),
}

Compliant Solution

if (variable == 0)
{
  doSomething();
}
else
{
  doSomethingElse();
}

var foo = variable == 0
  ? doSomething()
  : doSomethingElse();

Exceptions

For switch expressions with two arms, this rule generates an issue only when one of the arms has the discard (default) value.

Thus, the following code will not generate an issue

var foo = variable switch
{
  0 => doSomething(),
  1 => doSomethingElse(),
}
