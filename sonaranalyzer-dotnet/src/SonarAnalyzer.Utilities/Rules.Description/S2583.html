If a boolean expression cannot evaluate to anything but true, this expression is completely redundant. From time to time, such useless expression can be used on purpose to increase the readability of the code. But such expression deserves to be carefully reviewed to make sure that was the initial programmer's intent.

Noncompliant Code Example

if (a && b)
{
  if(a)   //a is always true
  {
  }
}

See

  • MISRA C:2004, 13.7 - Boolean operations whose results are invariant shall not be permitted.
  • MISRA C:2012, 14.3 - Controlling expressions shall not be invariant
  • MITRE, CWE-489 - Leftover Debug Code
  • MITRE, CWE-571 - Expression is Always True
  • CERT, MSC12-C. - Detect and remove code that has no effect or is never executed
  • CERT, MSC12-CPP. - Detect and remove code that has no effect
