ASP.NET is coming with a feature looking at HTTP requests to prevent potentially dangerous content to perform a cross-site scripting (XSS) attack. There is no reason to disable this mechanism even if it should not be the only one in place to prevent XSS attacks.

This rule raises an issue if System.Web.Mvc.ValidateInputAttribute(false) is set on a method.

Noncompliant Code Example

public class FooBarController : Controller
{
    [HttpPost]
    [ValidateInput(false)] // Noncompliant
    public ActionResult Purchase(string input)
    {
        return Foo(input);
    }
}

Compliant Solution

public class FooBarController : Controller
{
    [HttpPost]
    [ValidateInput(true)] // Compliant
    public ActionResult Purchase(string input)
    {
        return Foo(input);
    }
}
