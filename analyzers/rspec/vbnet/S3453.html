Why is this an issue?

A class with only private constructors canâ€™t be instantiated, thus, it seems to be pointless code.

Noncompliant code example

Public Class [MyClass]
    Private Sub New() // Noncompliant
        ' ...
    End Sub
End Class

Compliant solution

Public Class [MyClass]
    Public Sub New()
        ' ...
    End Sub
End Class

Exceptions

Classes that themselves access their private constructors (singletons or smart enums) are ignored. Classes with only static members are also ignored because they are covered by Rule {rule:vbnet:S1118}.
